
Полное руководство по протоколу сжатия информации для взаимодействия с ИИ


| **Категория**       | **Элемент**       | **Описание**                                                                                     | **Пример использования**                                                                                     | **Расшифровка примера**                                                                                     |
|---------------------|-------------------|-------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------|
| **Структурные маркеры** | `Req:`          | Требования к задаче или системе.                                                               | `Req: ✓✓Auth (OAuth2); ~Logging (ELK)`                                                                       | Обязательная авторизация через OAuth2, опциональное логирование через ELK.                                  |
|                     | `Stack:`         | Стек технологий.                                                                               | `Stack: Py/DRF/PG`                                                                                            | Python, Django REST Framework, PostgreSQL.                                                                  |
|                     | `Offer:`         | Предложение или решение.                                                                       | `Offer: RAG →prob-0.95; Stack: Qdrant/LLM`                                                                   | Предлагается использовать RAG с достоверностью 95%, стек: Qdrant и LLM.                                     |
|                     | `Opt:`           | Опциональные элементы.                                                                         | `Opt: Monitoring (Prometheus)`                                                                               | Опционально добавить мониторинг через Prometheus.                                                           |
|                     | `Arch:`          | Архитектурные решения.                                                                         | `Arch: µserv; →perf`                                                                                          | Микросервисная архитектура с акцентом на производительность.                                                |
|                     | `;`              | Разделение групп элементов.                                                                    | `Req: Auth; Logging; →perf`                                                                                   | Требования: авторизация, логирование, акцент на производительность.                                        |
|                     | `/`              | Альтернативы.                                                                                   | `DB: PG/MySQL`                                                                                                | База данных: PostgreSQL или MySQL.                                                                          |
|                     | `:`              | Ключ-значение.                                                                                  | `Metric: latency`                                                                                            | Метрика: задержка.                                                                                          |
|                     | `→`              | Причинно-следственная связь.                                                                  | `Batch(20-50 items/prompt) → ↓overhead ↑throughput`                                                          | Батч-обработка 20-50 элементов в промпте снижает накладные расходы и увеличивает пропускную способность.     |
|                     | `||`             | Параллельные процессы.                                                                         | `Task1 || Task2`                                                                                             | Задачи 1 и 2 выполняются параллельно.                                                                      |
| **Технические термины** | `PG`           | PostgreSQL.                                                                                     | `Stack: PG`                                                                                                   | Используется PostgreSQL.                                                                                    |
|                     | `DRF`           | Django REST Framework.                                                                         | `Stack: DRF`                                                                                                  | Используется Django REST Framework.                                                                         |
|                     | `µserv`         | Microservices.                                                                                  | `Arch: µserv`                                                                                                 | Архитектура: микросервисы.                                                                                  |
|                     | `ctx`            | Context.                                                                                        | `Req: ctx management`                                                                                         | Требование: управление контекстом.                                                                          |
|                     | `proc`           | Processing.                                                                                     | `Task: data proc`                                                                                             | Задача: обработка данных.                                                                                   |
|                     | `cfg`            | Config.                                                                                         | `File: app.cfg`                                                                                               | Файл: конфигурация приложения.                                                                              |
|                     | `PyDev`          | Python Developer.                                                                               | `Role: PyDev`                                                                                                 | Роль: разработчик на Python.                                                                                |
|                     | `FSDev`          | Full-Stack Developer.                                                                           | `Role: FSDev`                                                                                                 | Роль: фулстек-разработчик.                                                                                  |
|                     | `DevOps`         | DevOps Engineer.                                                                               | `Role: DevOps`                                                                                                | Роль: инженер DevOps.                                                                                       |
|                     | `2+y`            | 2+ years.                                                                                       | `Exp: 2+y`                                                                                                   | Опыт: 2+ года.                                                                                              |
|                     | `5K`             | 5000.                                                                                          | `Users: 5K`                                                                                                  | Пользователи: 5000.                                                                                         |
| **Спецсимволы**     | `↓`              | Снижает/уменьшает.                                                                            | `→ ↓overhead`                                                                                                 | Снижает накладные расходы.                                                                                  |
|                     | `↑`              | Увеличивает/повышает.                                                                         | `→ ↑throughput`                                                                                               | Повышает пропускную способность.                                                                            |
|                     | `✓`              | Требуется/обязательно.                                                                        | `Req: ✓Auth`                                                                                                 | Обязательная авторизация.                                                                                  |
|                     | `✓✓`            | Критическое требование.                                                                       | `Req: ✓✓Backup`                                                                                              | Критически важно: резервное копирование.                                                                   |
|                     | `~`              | Примерно/около или опционально.                                                               | `Time: ~1h; Opt: ~Logging`                                                                                   | Время: около 1 часа; опционально: логирование.                                                              |
|                     | `@`              | Локация/место.                                                                                | `Event @Minsk`                                                                                               | Мероприятие в Минске.                                                                                       |
|                     | `⚠`              | Предупреждение.                                                                               | `Status: ⚠Memory leak`                                                                                         | Статус: предупреждение о утечке памяти.                                                                     |
|                     | `✅`              | Подтверждено/работает.                                                                        | `Test: ✅Passed`                                                                                              | Тест: пройден успешно.                                                                                       |

|                     | `🔄`             | В процессе/нужно обновить.                                                                    | `Task: 🔄Refactor`                                                                                             | Задача: рефакторинг в процессе.                                                                             |

|                     | `→no-safe`       | Убирает смягчающие наречия.                                                                   | `→no-safe; Answer: Yes`                                                                                      | Ответ: "Да" (без "возможно" или "наверное").                                                                |

|                     | `→byte-X`        | Лимит токенов на ответ.                                                                       | `→byte-20; Answer: Short`                                                                                    | Ответ не более 20 токенов.                                                                                  |
|                     | `→spo`           | Только SPO-триплеты.                                                                         | `→spo; Data: X→Y→Z`                                                                                          | Данные в формате Subject-Predicate-Object.                                                                  |
|                     | `→tab`           | Вывод в виде таблицы.                                                                         | `→tab; Metrics: CPU/RAM`                                                                                     | Метрики CPU и RAM в виде таблицы.                                                                           |
|                     | `→code`          | Чистый код без пояснений.                                                                     | `→code; Function: def foo()`                                                                                 | Только код функции `foo`.                                                                                   |
|                     | `→math`          | LaTeX-формулы.                                                                               | `→math; Formula: E=mc²`                                                                                     | Формула в формате LaTeX.                                                                                    |
|                     | `→steps`         | Нумерованные шаги.                                                                            | `→steps; Task: Install`                                                                                       | Инструкция по установке в виде шагов: 1., 2., 3.                                                           |
|                     | `→arrow`         | Стрелочные списки.                                                                            | `→arrow; Items: A→B→C`                                                                                        | Элементы в виде стрелочного списка.                                                                         |
|                     | `→eng`           | Force-English.                                                                                | `→eng; Explain: Token`                                                                                        | Объяснение термина "Token" на английском.                                                                    |
|                     | `→ru`            | Force-Russian.                                                                                | `→ru; Explain: Токен`                                                                                        | Объяснение термина "Токен" на русском.                                                                       |
|                     | `→ans`           | One-line ответ.                                                                               | `→ans; Question: Capital of Belarus?`                                                                         | Ответ: "Минск" (без пояснений).                                                                              |
|                     | `→why`           | Только причины/доказательства.                                                               | `→why; Issue: Slow API`                                                                                       | Причины медленной работы API.                                                                               |
|                     | `→rev`           | Сначала вывод, потом объяснение.                                                             | `→rev; Result: 42; →why`                                                                                     | Сначала ответ "42", затем объяснение.                                                                       |
|                     | `→time`          | Timestamp для фактов.                                                                        | `→time; Fact: Updated 2026-04-17`                                                                             | Факту присвоен timestamp.                                                                                   |
|                     | `→risk`          | Риск-оценка (low/mid/high).                                                                   | `→risk; Action: Deploy`                                                                                      | Оценка рисков развёртывания.                                                                                |
|                     | `→cost`          | Подсчёт затрат (USD/токены).                                                                  | `→cost; Task: API calls`                                                                                     | Расчёт стоимости вызовов API.                                                                               |

**Примеры комплексных запросов:**

1. **Анализ производительности микросервиса:**
   **Вход:**
   "Проанализировать падение производительности микросервиса на Python с PostgreSQL за последние 7 дней. Сравнить метрики до и после обновления. Вывести данные в виде таблицы, указать риски и предложить решения. Ответ должен быть техническим, без самореференций, и умещаться в 50 токенов."
   **Выход:**
   `Ops: →perf; Stack: Py/µserv/PG; Metric: latency; →trend; Period: 7d; →cmp; →tab; →risk; →tech; →byte-50; →no-meta`

2. **Запрос на ревью кода:**
   **Вход:**
   "Нужно провести ревью pull request #42 с изменениями в архитектуре авторизации. Ответственный — фулстек-разработчик. Ответ должен быть кратким и без самореференций."
   **Выход:**
   `Dev: →review; Code: PR#42; Topic: Auth; →team; Owner: @FSDev; →ans; →no-meta`

3. **Сравнение инструментов:**
   **Вход:**
   "Сравнить PostgreSQL и MySQL по производительности и стоимости для проекта с 10K пользователей. Вывести результаты в виде таблицы."
   **Выход:**
   `Biz: →cmp; DB: PG/MySQL; Metric: perf/cost; Users: 10K; →tab`
