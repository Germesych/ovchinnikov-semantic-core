**Complete Guide to the Information Compression Protocol for AI Interaction**

---

| **Category**          | **Element**       | **Description**                                                                                     | **Usage Example**                                                                                     | **Example Decoding**                                                                                     |
|-----------------------|-------------------|-----------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------|
| **Structural Markers** | `Req:`           | Requirements for a task or system.                                                                | `Req: ✓✓Auth (OAuth2); ~Logging (ELK)`                                                                | Mandatory OAuth2 authentication, optional ELK logging.                                                  |
|                       | `Stack:`         | Technology stack.                                                                                 | `Stack: Py/DRF/PG`                                                                                     | Python, Django REST Framework, PostgreSQL.                                                              |
|                       | `Offer:`         | Proposal or solution.                                                                               | `Offer: RAG →prob-0.95; Stack: Qdrant/LLM`                                                             | Suggested RAG with 95% confidence, stack: Qdrant and LLM.                                                |
|                       | `Opt:`           | Optional elements.                                                                                 | `Opt: Monitoring (Prometheus)`                                                                       | Optional addition of Prometheus monitoring.                                                             |
|                       | `Arch:`          | Architectural decisions.                                                                           | `Arch: µserv; →perf`                                                                                  | Microservice architecture with a focus on performance.                                                 |
|                       | `;`              | Separation of element groups.                                                                      | `Req: Auth; Logging; →perf`                                                                            | Requirements: authentication, logging, focus on performance.                                            |
|                       | `/`              | Alternatives.                                                                                       | `DB: PG/MySQL`                                                                                         | Database: PostgreSQL or MySQL.                                                                         |
|                       | `:`              | Key-value.                                                                                          | `Metric: latency`                                                                                      | Metric: latency.                                                                                         |
|                       | `→`              | Cause-and-effect relationship.                                                                     | `Batch(20-50 items/prompt) → ↓overhead ↑throughput`                                                    | Batch processing 20-50 items per prompt reduces overhead and increases throughput.                     |
|                       | `||`             | Parallel processes.                                                                               | `Task1 || Task2`                                                                                       | Tasks 1 and 2 run in parallel.                                                                          |
| **Technical Terms**   | `PG`             | PostgreSQL.                                                                                         | `Stack: PG`                                                                                           | Uses PostgreSQL.                                                                                         |
|                       | `DRF`            | Django REST Framework.                                                                             | `Stack: DRF`                                                                                          | Uses Django REST Framework.                                                                             |
|                       | `µserv`          | Microservices.                                                                                      | `Arch: µserv`                                                                                         | Architecture: microservices.                                                                             |
|                       | `ctx`             | Context.                                                                                            | `Req: ctx management`                                                                                 | Requirement: context management.                                                                         |
|                       | `proc`            | Processing.                                                                                         | `Task: data proc`                                                                                     | Task: data processing.                                                                                  |
|                       | `cfg`             | Config.                                                                                             | `File: app.cfg`                                                                                       | File: application config.                                                                               |
|                       | `PyDev`           | Python Developer.                                                                                   | `Role: PyDev`                                                                                         | Role: Python developer.                                                                                  |
|                       | `FSDev`           | Full-Stack Developer.                                                                               | `Role: FSDev`                                                                                         | Role: full-stack developer.                                                                              |
|                       | `DevOps`          | DevOps Engineer.                                                                                   | `Role: DevOps`                                                                                         | Role: DevOps engineer.                                                                                   |
|                       | `2+y`            | 2+ years.                                                                                           | `Exp: 2+y`                                                                                             | Experience: 2+ years.                                                                                   |
|                       | `5K`             | 5000.                                                                                               | `Users: 5K`                                                                                           | Users: 5000.                                                                                             |
| **Special Symbols**  | `↓`              | Reduces/decreases.                                                                                 | `→ ↓overhead`                                                                                          | Reduces overhead.                                                                                         |
|                       | `↑`              | Increases/improves.                                                                                 | `→ ↑throughput`                                                                                        | Improves throughput.                                                                                     |
|                       | `✓`              | Required/mandatory.                                                                                | `Req: ✓Auth`                                                                                           | Mandatory authentication.                                                                                |
|                       | `✓✓`            | Critical requirement.                                                                               | `Req: ✓✓Backup`                                                                                        | Critically required: backup.                                                                             |
|                       | `~`              | Approximately/optional.                                                                             | `Time: ~1h; Opt: ~Logging`                                                                             | Time: ~1 hour; optional: logging.                                                                          |
|                       | `@`              | Location/place.                                                                                    | `Event @Minsk`                                                                                         | Event in Minsk.                                                                                           |
|                       | `⚠`              | Warning.                                                                                             | `Status: ⚠Memory leak`                                                                                 | Status: memory leak warning.                                                                             |
|                       | `✅`              | Confirmed/works.                                                                                    | `Test: ✅Passed`                                                                                        | Test: passed successfully.                                                                               |
|                       | `❌`              | Error/does not work.                                                                                 | `Build: ❌Failed`                                                                                        | Build: failed.                                                                                            |
|                       | `🔄`             | In progress/needs update.                                                                           | `Task: 🔄Refactor`                                                                                     | Task: refactoring in progress.                                                                            |
| **Modifiers**         | `→no-meta`       | Removes AI self-references.                                                                         | `→no-meta; Explain: LLM`                                                                               | Explain LLM without mentioning "I as an AI".                                                           |
|                       | `→no-safe`       | Removes softening adverbs.                                                                          | `→no-safe; Answer: Yes`                                                                                | Answer: "Yes" (without "possibly" or "maybe").                                                           |
|                       | `→prob-N`        | Probability assessment (0-1).                                                                       | `→prob-0.93; Fact: True`                                                                               | Assigns 93% probability to the fact.                                                                     |
|                       | `→byte-X`        | Token limit for the response.                                                                       | `→byte-20; Answer: Short`                                                                               | Response limited to 20 tokens.                                                                           |
|                       | `→spo`           | Only SPO triples.                                                                                   | `→spo; Data: X→Y→Z`                                                                                     | Data in Subject-Predicate-Object format.                                                                 |
|                       | `→tab`           | Output as a table.                                                                                   | `→tab; Metrics: CPU/RAM`                                                                                | CPU and RAM metrics as a table.                                                                           |
|                       | `→code`          | Pure code without explanations.                                                                     | `→code; Function: def foo()`                                                                            | Only the code for function `foo`.                                                                         |
|                       | `→math`          | LaTeX formulas.                                                                                     | `→math; Formula: E=mc²`                                                                                 | Formula in LaTeX format.                                                                                  |
|                       | `→steps`         | Numbered steps.                                                                                     | `→steps; Task: Install`                                                                                 | Installation instructions as steps: 1., 2., 3.                                                           |
|                       | `→arrow`         | Arrow lists.                                                                                         | `→arrow; Items: A→B→C`                                                                                  | Items as an arrow list.                                                                                   |
|                       | `→eng`           | Force-English.                                                                                       | `→eng; Explain: Token`                                                                                   | Explain the term "Token" in English.                                                                     |
|                       | `→ru`            | Force-Russian.                                                                                       | `→ru; Explain: Токен`                                                                                   | Explain the term "Токен" in Russian.                                                                      |
|                       | `→ans`           | One-line answer.                                                                                     | `→ans; Question: Capital of Belarus?`                                                                  | Answer: "Minsk" (no explanations).                                                                       |
|                       | `→why`           | Only reasons/evidence.                                                                              | `→why; Issue: Slow API`                                                                                  | Reasons for slow API.                                                                                     |
|                       | `→rev`           | Result first, then explanation.                                                                     | `→rev; Result: 42; →why`                                                                                 | First answer "42", then explanation.                                                                       |
|                       | `→time`          | Timestamp for facts.                                                                                | `→time; Fact: Updated 2026-04-17`                                                                        | Fact assigned a timestamp.                                                                               |
|                       | `→risk`          | Risk assessment (low/mid/high).                                                                     | `→risk; Action: Deploy`                                                                                 | Risk assessment for deployment.                                                                          |
|                       | `→cost`          | Cost calculation (USD/tokens).                                                                       | `→cost; Task: API calls`                                                                                 | Cost calculation for API calls.                                                                          |
|                       | `→cmp`           | Before/after comparison.                                                                            | `→cmp; Metric: Latency`                                                                                  | Comparison of latency before and after optimization.                                                     |
|                       | `→data`          | Only data (no analysis).                                                                             | `→data; →tab; Metrics: CPU`                                                                              | CPU metrics table without analysis.                                                                      |
|                       | `→stat`          | Statistical analysis.                                                                               | `→stat; Metric: Response time`                                                                           | Response time analysis (mean, median).                                                                   |
|                       | `→trend`         | Trend analysis.                                                                                     | `→trend; Period: 7d`                                                                                    | Trends over the last 7 days.                                                                               |
|                       | `→outlier`       | Outlier detection.                                                                                   | `→outlier; Threshold: 3σ`                                                                                 | Outlier detection with a 3-sigma threshold.                                                              |
|                       | `→corr`          | Correlation analysis.                                                                               | `→corr; Vars: X/Y`                                                                                       | Correlation analysis between X and Y.                                                                    |
|                       | `→arch`          | Focus on architecture.                                                                               | `→arch; System: µserv`                                                                                  | Focus on microservice architecture.                                                                       |
|                       | `→perf`          | Focus on performance.                                                                               | `→perf; Metric: RPS`                                                                                     | Optimization of requests per second.                                                                      |
|                       | `→now`           | Urgent request.                                                                                       | `→now; Fix: Bug`                                                                                         | Fix the bug immediately.                                                                                  |
|                       | `→later`         | Can be postponed.                                                                                     | `→later; Task: Refactor`                                                                                 | Refactoring can be postponed.                                                                              |
|                       | `→deadline:YYYY-MM-DD` | Deadline.                                                                               | `→deadline:2026-04-20; Task: Release`                                                                  | Release deadline: April 20, 2026.                                                                         |
|                       | `→team`          | Mention of roles or responsible parties.                                                            | `→team; Owner: @FSDev`                                                                                   | Responsible: full-stack developer.                                                                        |
|                       | `→review`        | Request for review.                                                                                   | `→review; Code: PR#42`                                                                                   | Request for review of pull request #42.                                                                   |
|                       | `→meet`          | Proposal to discuss offline.                                                                          | `→meet; Topic: Architecture`                                                                             | Discuss architecture offline.                                                                             |
|                       | `→api:[name]`    | API request.                                                                                         | `→api:Stripe; Task: Payment`                                                                             | Integration with Stripe API for payments.                                                                 |
|                       | `→db:[name]`     | Database operation.                                                                                   | `→db:PG; Query: SELECT * FROM users`                                                                     | Query to PostgreSQL.                                                                                       |
|                       | `→tool:[name]`   | Tool usage.                                                                                            | `→tool:Docker; Task: Build`                                                                                | Build using Docker.                                                                                         |
| **Domain Prefixes**    | `ML:`           | Machine Learning.                                                                                     | `ML: →perf; Model: LLM`                                                                                   | Performance optimization for LLM model.                                                                  |
|                       | `Sec:`           | Security.                                                                                             | `Sec: ↓attack surface; →why`                                                                             | Reduce attack surface: reasons.                                                                          |
|                       | `Biz:`           | Business logic/analytics.                                                                            | `Biz: ROI ↑20%; →cmp`                                                                                    | 20% ROI increase: comparison.                                                                              |

|                       | `UX:`            | User Experience.                                                                                       | `UX: ↓friction; →why`                                                                                     | Reduce friction: reasons.                                                                                 |
|                       | `Ops:`           | DevOps/Infrastructure.                                                                                 | `Ops: →perf; Metric: Uptime`                                                                              | Uptime optimization.                                                                                       |
| **Tone**              | `→formal`        | Formal style.                                                                                         | `→formal; Report: Quarterly`                                                                              | Report in a formal style.                                                                                 |
|                       | `→casual`        | Informal style.                                                                                       | `→casual; Explain: Blockchain`                                                                            | Simple explanation of blockchain.                                                                         |
|                       | `→tech`          | Highly technical style.                                                                               | `→tech; Topic: Kubernetes`                                                                                 | Technical explanation of Kubernetes.                                                                      |

**Complex Query Examples:**

1. **Microservice Performance Analysis:**
   **Input:**
   "Analyze the performance drop of a Python microservice with PostgreSQL over the last 7 days. Compare metrics before and after the update. Output the data as a table, indicate risks, and suggest solutions. The response should be technical, without self-references, and fit within 50 tokens."
   **Output:**
   `Ops: →perf; Stack: Py/µserv/PG; Metric: latency; →trend; Period: 7d; →cmp; →tab; →risk; →tech; →byte-50; →no-meta`

2. **Code Review Request:**
   **Input:**
   "Conduct a review of pull request #42 with changes to the authentication architecture. The responsible party is the full-stack developer. The response should be concise and without self-references."
   **Output:**
   `Dev: →review; Code: PR#42; Topic: Auth; →team; Owner: @FSDev; →ans; →no-meta`

3. **Tool Comparison:**
   **Input:**
   "Compare PostgreSQL and MySQL in terms of performance and cost for a project with 10K users. Output the results as a table."
   **Output:**
   `Biz: →cmp; DB: PG/MySQL; Metric: perf/cost; Users: 10K; →tab`
